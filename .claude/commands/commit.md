# Skill: 提交代码

当用户说"提交"、"帮我提交"、"commit"或类似表达时，执行以下流程：

## 执行步骤

1. **查看变更**

   ```bash
   git status && git diff --stat
   ```

2. **如果工作区干净**
   - 直接告诉用户没有可提交的变更

3. **如果有待提交内容**
   - 读取变更文件的 diff 内容
   - 分析本次提交的变更内容
   - 总结变更摘要

4. **更新 ARCHITECTURE.md**
   - 读取当前文档
   - 在更新日志中添加新条目，格式：`| YYYY-MM-DD | 变更摘要 |`
   - 如果项目的功能、架构有调整，需要更新文档对应的内容
   - 更新"最后更新时间"

5. **提交代码**

   ```bash
   git add .
   git commit -m "<变更摘要>"
   ```

6. **显示结果**
   - 告知用户提交成功
   - 显示 commit hash 和变更摘要

## 注意事项

- 如果工作区有未暂存的变更，先 `git add .` 暂存
- 提交信息应该简洁描述本次变更内容
- 确保 ARCHITECTURE.md 也一起提交
